# 주요 작업 리뷰 

작업 - 토크나이저: Mecab 대신 OKT를 사용하여 한글 토크나이징 수행 (Mecab에서 지속적인 에러 발생)
리뷰 - Mecab 설치 오류의 대안으로 OKT를 선택한 것은 매우 합리적입니닷.
  
작업 - 데이터 전처리: 중복 제거 시 한글-영어 쌍의 매칭 문제로 인해 중복 제거를 하지 않음
리뷰 - 중복 데이터는 모델이 특정 문장을 암기하게 하여 과적합(Overfitting)을 유발할 수 있지만 매칭 문제 때문에 어쩔 수 없는 것 같습니다. 
  
작업 - 데이터셋: 총 6만 개의 문장으로 학습 진행 (초기 결과가 좋지 않아 데이터 양 증가)
리뷰 - 성능 저하를 해결하기 위해 데이터 양을 6만 개로 증설한 것은 합리적 판단 같습니다. 

작업 - 모델 구조: LSTM 3층 구조로 인코더와 디코더 모두 구성
리뷰 - LSTM을 3층으로 깊게 쌓아 표현력을 높이는 중요한 결정인 것 같습니다. 

작업 - 학습 환경: LMS 환경에서 CPU 사용으로 인해 학습 시간이 6~7시간 소요
리뷰 - CPU 학습은 실험 주기를 역시나 매우 느리게 만드네요.

작업 - - 학습 설정: 에포크 10회 (20회는 너무 많아서 축소), 학습 시간 1시간 30분 이상
리뷰 - 에포크를 줄여도 학습 시간이 1시간 30분!!

작업 - 성능 개선: OKT 사용, 배치 사이즈 증가, 레이어 수 증가로 번역 결과가 개선됨
리뷰 - 토크나이저 변경과 하이퍼파라미터(배치 사이즈, 레이어 수) 조정을 통해 실제 개선을 이끌어낸 점은 실험 설계로서 훌륭한 접근인 것 같습니다. 

작업 - 번역 결과: "오바마는 대통령이다"에 불필요한 단어 추가, "7명의 사망자" → "20명"으로 오번역 등 문제 발생
리뷰 - 숫자 오번역이나 환각(Hallucination) 현상은 전형적인 학습 부족의 증상이라고 들었습니다.
